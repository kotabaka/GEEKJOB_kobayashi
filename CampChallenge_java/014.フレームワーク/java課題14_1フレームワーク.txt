課題：javaのフレームワーク
javaで有名、もしくは興味のあるフレームワークを3つ調べ特徴をまとめる。

①JSF
・公式サイトの紹介文
　　JSFはJava EE（Java Platform, Enterprise Edition）という環境で動作
　　する標準的なコンポーネント志向（機能ごとに部品として扱う考え方）の
　　ユーザーインターフェイスフレームワークです。JSFはJavaEEに含まれて
　　いるので、新しくライブラリなどをプロジェクトに追加しなくても使うこ
　　とができます。
・MVCモデルを採用したフレームワーク
・表示する画面に対応するサーバーサイドのクラスによって処理が行われる
　「コンポーネントベースフレームワーク」
・JSFのViewは「XHTML」というXML形式のHTMLで実装されブラウザでそのまま
　表示されるのでデザインの確認がしやすい
・開発チームとデザインチームで分業しやすい

②Spring　Framework
・公式サイトの紹介文
　スプリングフレームワークは近代的なJavaの業務アプリ開発の包括的なプロ
　グラミング環境を供給します。　
・オブジェクトの関連付け、生成、登録などを管理する「Coreモジュール」、
　JavaBeansへのアクセス機能を提供する「Context モジュール」、MVCモデ
　ルでの開発を提供する「Web MVCモジュール」など7種類のモジュールで構
　成されている
・webアプリケーションだけでなく、javaプラットフォームで動作する様々な
　アプリケーションに適用できる
・世界的に人気がありすぐにすたれることはなさそう
・万能なフレームワークで様々なシステム構築に対応可能

③Play　Framework
・公式サイトの紹介文
　Play は、現代の web アプリケーション開発に必要なコンポーネント及び 
　API を統合した生産性の高い Java と Scala の web アプリケーションフ
　レームワークです。Play の特徴は、ライトウェイト、ステートレス、web
　フレンドリーなアーキテクチャであること、機能予測のしやすさです。
　また、Iteratee IO をベースにしたリアクティブモデルのおかげで、
　スケーラブルなアプリケーションでも CPU、メモリ、スレッドなどのリソ
　ース消費が最小限になっています。
・javaだけでなくScalaというjavaと親和性の高い言語でも使用することができる。
・軽量さと高い生産性が特徴
・動作時に使用するCPUリソースやメモリ使用量も少ないためハードウェアの
　負担を抑えたシステム構築ができる
